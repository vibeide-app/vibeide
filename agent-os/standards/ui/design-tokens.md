# CSS Design Token System

All styling uses CSS custom properties from `:root`. Never hardcode colors, sizes, or transitions.

## Token Categories

**Colors** (theme-driven):
`--bg`, `--bg-deep`, `--fg`, `--fg-dim`, `--fg-faint`, `--border`, `--accent`, `--surface`, `--surface-hover`, `--error`, `--success`, `--warning`

**Spacing** (fixed scale):
`--space-1` (4px) through `--space-8` (32px)

**Typography** (fixed scale):
`--font-xs` (10px) through `--font-lg` (16px), `--leading-tight/normal/relaxed`

**Transitions**:
`--transition-fast` (0.1s), `--transition-normal` (0.15s), `--transition-smooth` (0.25s cubic-bezier)

## Rules

- Use tokens, never raw values in CSS
- Themes override color tokens via JS (`applyTheme` sets root style properties)
- Spacing/typography tokens are fixed across themes
- Derived tokens (`--accent-dim`, `--border-subtle`) auto-generated by `enrichChrome()`

## Why

- Instant theme switching without JS re-renders
- No framework or build-step dependency
- Single source of truth for the visual language
